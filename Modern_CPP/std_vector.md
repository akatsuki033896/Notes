> C++ 标准库五大件：容器container / 迭代器iterator / 分配器allocator / 输入输出流stream / 仿函数function / 算法algorithm

`vector `的功能是长度可变的数组，里面的数据存储在堆上

# `vector` 容器的成员函数

## 声明和成员函数 `size()`

`vector` 是一个模板类，第一个模板参数是数组里元素的类型，例如 `vector<int> a`，`vector<int> a(4)` 指定长度为 `4`

通过 `a.size()` 获取到长度为 `4`

## 访问

### `[]`：运算符重载

返回容器内元素引用。通过 `vector` 重载好的运算符 `[]` 来访问内部元素，`a[0]` 访问第 `0` 个，以此类推。但是 `[]` 运算符在索引超出数组大小时并不会直接报错，这是为了性能的考虑。如果你不小心用 `[] `访问了越界的索引，可能会覆盖掉别的变量导致程序行为异常，或是访问到操作系统未映射的区域导致崩溃。

可以用来赋值 `a[i]=100`

### 成员函数 `at()`

返回容器内元素引用。为了防止越界可以使用 `a.at(i)` 替代 `a[i]`，`at()` 会检测索引 `i` 是否越界，如果他发现 `i >= a.size()` 则会抛出异常 `std::out_of_range` 让程序提前终止（或者被 `try-catch` 捕获），配合调试器可以很快速地定位到出错点。不过 `at()` 需要额外检测下标是否越界，虽然更安全方便调试，但和 `[]` 相比有一定性能损失。同样可以写入。

## 构造函数

可使用初始化列表构造 `vector<int> a = {1, 2, 3, 4}` 或 `vector<int> a{1, 2, 3, 4}` 

### 显式构造函数

如果需要元素全部为 `0` 的数组，必须用`()` 才能显式构造，例如  `vector<int> a(4)` 构造全 `0` 长度为 `4` 的数组。这个显式构造函数，默认会把所有元素都初始化为 `0`，若指定第二个参数可以把整个数组初始化为指定的值，例如 `vector<int> a(4, 233)` 等价于 `vector<int> a = {233, 233, 233, 233}`

## 成员函数`resize()`

通过 `resize` 函数设置大小，例如 `a.resize(4)` 设置为 `4`，`a.resize(4, 233)` 用声明的值 `233` 填充所有新增的元素，原有元素保持不变。若不声明第二个值则默认填充 `0`。若 `resize` 后的容器大小比原有的小，则删除多余的元素。

## 成员函数 `clear()`

清空该数组。如果需要把原数组前面的部分也填充上 `0`，可以先 `clear` 再 `resize`，这是一个常见的组合。

`clear` 相当于 `resize(0)`，所以他也不会实际释放掉内存，容量还是摆在那里，`clear` 仅仅只是把 `size` 标记为 `0` 而已，这可能导致在低端平台上内存告急，这是因为实际容量还在并没有释放。

## 成员函数 `push_back()` `pop_back()`

`push_back()` 在数组的末尾追加一个数，C++11新增定义 `void push_back(int &&val)`

`pop_back()` 在数组的末尾删除一个数

```cpp
vector<int> a = {1, 2};
a.push_back(3); // 等价于 vector<int> a = {1, 2, 3}
a.pop_back(); // 等价于 vector<int> a = {1, 2}
```

## 成员函数 `front()` `back()`

`front()` 获取首个元素的引用 `a[0]`，`back()` 获取末尾元素的值

## 成员函数 `data()`

`data()` 会返回指向数组中首个元素的指针，等价于 `&a[0]`。由于 `vector` 是连续存储的数组，因此只要得到了首地址，下一个元素的地址只需指针 `+1` 即可。因为指针的 `p[i]` 相当于 `*(p + i)`，因此可以把 `data()` 返回的首地址指针当一个数组来访问

```cpp
vector<int> a = {1, 2, 3, 4, 5}
int *p = a.data();
// p[0] = 1, p[1] = 2 ...
```

`data()` 返回的首地址指针，通常配合 `size()` 返回的数组长度一起使用，用他来获取一个 C 语言原始指针 `int *`，很方便用于调用 C 语言的函数和 API 等，同时还能享受到 `vector` 容器 RAII 的安全性

```cpp
int *p = a.data();
int n = a.size();
memset(p, -1, sizeof(int) * n);
```

### 和 `resize()` 配合使用

当 `resize` 的目标长度大于原有的容量时，就需要重新分配一段更大的连续内存，并把原数组长度的部分移动过去，多出来的部分默认用 `0` 来填充。这就导致元素的地址会有所改变，从而过去 `data` 返回的指针以及所有的迭代器对象，都会失效。

当 `resize` 的目标长度小于原有的容量时，不需要重新分配一段连续的内存也不会造成元素的移动所以指向元素的指针不会失效。他只是会把数组的长度标记为新长度，后面空闲出来那一段内存不会释放掉，继续留在那里，直到 `vector` 对象被解构。

重新 `resize` 到原有尺寸不需要重新分配内存，也就不会移动元素导致指针失效。

## 成员函数 `capacity()`

查询已经分配内存的大小，即最大容量，相对的 `size()` 返回的其实是已经存储了数据的数组长度

### `resize()` 的优化策略

1. 当 `resize` 指定的新长度一个超过原来的最大容量时时，就会重新分配一段更大容量的内存来存储数组，只有这时才会移动元素的位置（data 指针失效）
2. 为了减少重复分配的次数，当 `resize` 后的新尺寸变化较小时，则自动扩容至原尺寸的两倍，例如原大小为 `5` 时 `resize` 到 `7`，会扩充容量到 `10`，`resize(n) `的逻辑是扩容至 `max(n, capacity * 2)`。

## 成员函数 `reserve()` `shrink_to_fit()`

如果我们能预料到数组最终的大小，可以用 `reserve()` 函数预留一定的容量，这样之后就不会出现容量不足而需要动态扩容影响性能了，例如一开始预留了 `12` 容量，这样从 `5` 到 `12` 的时候就不必重新分配。此外，还要注意 `reserve` 时也会移动元素

```cpp
vector<int> a = {1,2,3,4,5}
a.reserve(12);
```

担心内存告急可以用 `shrink_to_fit`释放掉多余的容量，只保留刚好为 `size() `大小的容量，`shrink_to_fit` 会重新分配一段更小内存，他同样是会把元素移动到新内存中的，因此迭代器和指针也会失效。

### `push_back()` 的内存管理

由于不知道你究竟会推入多少个元素，`vector` 的初始容量是零，而 `push_back` 和 `resize` 一样，每次遇到容量不足时，都会扩容两倍，因此，如果早就知道要推入元素的数量，可以调用 `reserve` 先预留那么多的容量，等待接下来的推入，这样之后 `push_back` 时，就不会一次次地扩容两倍慢慢
成长，避免重新分配内存和移动元素，更高效

标准库这样允许数组大小和实际容量不同，这样 `push_back` 在容量不足的时候就可以一次性扩容两倍，只需重新分配 $logn$ 次，移动元素 $2n-1$ 次

### `clear()` 的内存管理

要真正释放掉内存，可以在 `clear` 之后再调用 `shrink_to_fit`，这样才会让容量也变成 `0`（这时 `vector` 的 `data` 会返回 `nullptr`）

# `vector` 容器的 RAII

1.  `vector` 会在离开作用域时，自动调用解构函数，释放内存，就不必手动释放了，更安全
2. vector 会在退出作用域时释放内存，这时候所有指向其中元素的指针，包括 `data()` 都会失效。因此如果你是在作用域内获取的 `data()` 指针，语句块外就无法访问了
3. ` }`  运算符标志着一个作用域的结束，在这里，他会调用所有身处其中的对象的解构函数。比如这里的 `vector`，他的解构函数会释放动态数组的内存，自动 `delete`

## 延续生命周期

1. 将作用域内的 `vector` 移到外面的作用域中，则获取的 `data()` 对数组的弱引用不会失效
2. 也可以移到全局（C++ 规定全局变量都会在进入 `main` 函数之前构造，`main` 函数返回之后解构）至于那个全局变量本身有没有被使用则无
   所谓（我们是通过首地址指针间接访问）他的存在只是为了延续生命周期，告知编译器什么时候能 `delete` 而已

# `vector` 容器的迭代器特征

## `begin()` 和 `end()`

`begin()` 获取指向第一个元素所在位置的迭代器，通过 `*a.begin() `来访问第一个元素

`end()` 获取指向**最后一个元素下一个位置**的迭代器（迭代器实际上是一个指针，不需要指定原来的容器本身，就能知道指定的位置）注意 `end()` 指向的位置不可用，访问最后一个元素必须用 `*(a.end() - 1) `

一对迭代器 `begin` 和 `end` 就标记了一个区间（range，C++20强化了它的概念）区间可以是一个容器的全部，例如 `{a.begin(), a.end()}`  也可以是一个容器的部分，例如 `{a.begin() + 1, a.end() - 1}` 相当于去头去尾后的列表，相当于 Python 中的 `a[1:-1]`

> 迭代器对象和容器本身的主要区别
>
> 迭代器不掌握生命周期，从而迭代器的拷贝是浅拷贝，方便传参。但也带来了缺点，因为迭代器是一个对原容器的弱引用，如果原容器解构或发生内存重分配，迭代器就会失效。

## 成员函数`insert()`

1. 第一个参数是要插入的位置（用迭代器表示），第二个参数则是要插入的值。例如 `a.insert(a.begin() + 3, 233)` 为把 `233` 插入第三个元素之前
2. 三个参数的重载 `a.insert(a.begin(), 4, 233)` 为把 `233` 插入到开头，插入 `4` 次
3. `a.insert(a.begin(), {1, 2, 3})` 插入一个初始化列表
4. 不支持 `vector` 本身作为参数，但支持自由选择对方容器的一个子区间（通过迭代器加减法）内的元素来插入，可以批量插入从来自另一个不同类型的容器，例如 `list<int>`，只要元素类型相等，且符合迭代器规范

```cpp
vector<int> a = {1, 2, 3, 4};
vector<int> b = {233, 666, 985};
a.insert(a.begin(), b.begin(), b.end()); // 把 b 插入在原先 a 元素之前
a.insert(a.end(), b.begin(), b.end()) // 把 b 插入在原先 a 元素之后
a.insert(a.begin() + 3, b.begin(), b.end()) // 把 b 插入在原先 a 元素中间
```

由于插入位置后方的元素要整体向后移动一格，因此复杂度为 $O(n)$，$n$​ 为 是从插入位置 pos 到数组末尾 end 的距离。为了高效尽量只往尾部插入元素。`insert` 的三个参数的重载会一次性批量让 `pos` 之后的元素移动 `n` 格，不存在反复移动 1 格的情况，最坏复杂度仍然是 $O(n)$，插入初始化列表也是 $O(n)$，

插入的对方的容器也可以是不同类型，最底线的要求是只要他的迭代器有 `++` 和 `*` 运算符即可，还可以是 C 的数组，传参时用 `std::begin` 和 `std::end` 这两个全局函数代替，返回 `c` 和 ` c + sizeof(c)/sizeof(c[0])`

```cpp
vector<int> a = {1, 2, 3, 4};
list<int>::iterator b = {233, 666, 985};
int c[] = {233, 666, 985}; // c 风格数组
```

## 成员函数 `assign()`

1. 把旧有的数组完全覆盖掉，变成一个新的数组，`a.assign(beg, end)`  基本和 `a = vector<int>(beg, end)` 等价，唯一的区别是后者会重新分配内存，而前者会保留原来的容量不会释放掉
2. 重载`a.assign(n, val)` 可以把 `vector `批量填满一个特定的值，重复的次数（长度）也是参数里指定，`a.assign(n, val)`  基本和 `a = vector<int>(n, val)` 等价，唯一的区别是后者会重新分配内存，而前者会保留原来的容量
3. 可接受初始化列表当参数，`a.assign({x, y, ...})` 和 `a = {x, y, ...}` 完全等价，都会保留原来的容量。而和 `a = vector<int>{x, y, ...}` 就不等价，这个会重新分配内存

```cpp
vector<int> a = {1, 2, 3, 4, 5, 6};
int b[] = {233, 666, 985, 211};
a.assign(std::begin(b), std::end(b)); // a = {233, 666, 985, 211}
a.assign(4, 233); // a = {233, 233, 233, 233}
a.assign({233, 666, 985, 211});
```

## 成员函数 `erase()`

删除指定位置的一个元素（通过迭代器指定）复杂度最坏情况是删除第一个元素 $O(n)$，如果删的是最后一个元素则复杂度为 $O(1)$（ erase 会移动 pos 之后的那些元素）

```cpp
a.erase(a.begin()) // 删除第一个元素
a.erase(a.end() - 1) // 删除最后一个元素
a.erase(a.begin() + 2) // 删除第三个元素
a.erase(a.end() - 2) // 删除倒数第二个元素
```

可以指定两个迭代器作为参数，表示把这个区间内的对象都删除，注意第一个迭代器指向的元素必须在第二个迭代器之前，不能反过来指定。批量删除的最坏复杂度依然是 $O(n)$ 的。不过这里两个作为 `erase` 参数的迭代器必须是自己这个对象的迭代器，不能是其他容器的，这点和 `insert `不一样。

```cpp
a.erase(a.begin() + 1, a.begin() + 3) // 删除第二个第三个
```
