# `auto`：自动类型推导(C++11)

## 定义变量

使用 `auto` 定义的变量，类型根据等号右边的值确定，避免重复写类型名

```cpp
struct MyClassWithVeryLongName {};
auto p = std::make_shared<MyClassWithVeryLongName>();
```

**值没有初始化则不能使用** `auto`，编译器强制提醒你赋值，类的成员也不可以定义为 `auto`

## 用作函数返回类型

```cpp
auto func() {
  return std::make_shared<MyClassWithVeryLongName>();
}
```

自动推导为 `return` 右边的类型

1. 函数有多条 `return` 的时候所有语句的返回类型必须一致
2. 没有 `return` 时推导为 `void`
3. 声明和实现分离时不能声明为 `auto` 例如 `auto func()`

# 引用

引用是 C++ 的特殊类型，本质是指针，修改一个引用就是修改了原来的对象，减少了 C语言中 `&` 和 `*` 的麻烦，例如 `int &ref = x` 等价于 `int *ref = &x`

## `const&` ：常引用

`const` 使得 `ref` 不能被写入值，更加安全，编译器可以放心的自动优化

```cpp
int main() {
  int x = 233;
  int const &ref = x;
  // ref = 42; 不能赋值
}
```

## `auto&` 定义引用

定义引用：`auto &ref = x`

定义常引用：`auto const &ref = x`

## 函数返回引用

函数返回类型也可以是 `auto&` 和 `auto const&`

```cpp
auto &product_table() {
  static std::map<std::string, int> instance;
  return instance;
}
```

## 左值和右值

引用又称为左值，对应一个长时间存在于内存中的变量

右值是一个表达式，代表计算过程中临时生成的中间变量

例如对于 `int a; int *p`，左值类型为 `int &`，左值为 `a`，右值类型为 `int &&`，右值为 `a + 1`

# `decltype` 推导类型

## 获取变量类型

```cpp
int a = 1; // int
int &b = a; // int&
int const &c = a; // int const &
decltype(a) d = a;
```

使用 `decltype(变量名)` 获取变量定义时候的类型

## 获取表达式类型

通过 `decltype(表达式)` 获取表达式类型

## `decltype(auto)`：万能推导

如果一个表达式，不知道他是可变引用 / 常引用 / 右值引用 / 普通的值，但就是想要定义一个和表达式返回类型的变量，可以用 `decltype(auto) p = func()` 自动推导为 `func()` 的返回类型，等价于 `decltype(func()) p = func()`，在代理模式中用于完美转发函数返回值

## `using`：创建类型别名

和 `typedef` 一样，`typedef std::vector<int> Veclnt` 等价于 `using Veclnt = std::vector<int>`

## `decltype` 的应用

```cpp
template<class T1, class T2>
auto add(std::vector<T1> const &a, std::vector<T2> const &b) {
  using T0 = decltype(T1{} + T2{});
  std::vector<T0> ret;
  for (size_t i = 0; i < std::min(a.size(), b.size()); i++) {
    ret.push_back(a[i] + b[i]);
  }
  return ret;
}
```

实现了不同类型容器逐元素相加的功能
