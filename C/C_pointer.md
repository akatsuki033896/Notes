# 计算机组成原理

复习计算机组成原理中数的表示

## word / byte / bit

每个存储 0 和 1 的空间是位(bit)，8个 bit 打包成一个字节(byte)，1 byte 可以表示 $0\sim255$，字节在 C 中表示为 `unsigned char` 类型，16位计算机将两个字节并成一个字(word) 就是 C 中的 `unsigned short`，同理现代计算机是 32位，把 4 个字节并成一个字，可以表示 $0 \sim 2^{32}-1$，把 8 个字节并成一个字，可以表示 $0 \sim 2^{64}-1$。计算机字的长度是一次能做运算的数的位的大小，超过该位数就要使用特殊指令。

### 内存地址的表示

字决定了能访问内存地址的范围，决定了寄存器的大小。内存是一维排列的，假如内存容量是 $65536$ byte，那内存地址实际上就是一个从 $0$ 到 $65535$ 范围的整数，也就是两个字节组成的字。使用 `lscpu` 命令可以查看处理器相关信息。

## 有符号整数 / 无符号整数

有符号整数分为原码 / 反码 / 补码 / 移码，牺牲一个最高位表示符号位，剩下表示数值，最常用的补码表示法让有符号整数可以表示 $-128 \sim 127$，负数范围比正数大是因为要防止表示出 $-0$ 

# C 语言的整数类型

上文已经提过 `unsigned` 类型，只能表示正数，接下来看有符号整数类型

| 类型        | Unix 32位 | Unix 64位 | Windows 32位 | Windows 64位 |
| ----------- | --------- | --------- | ------------ | ------------ |
| `char`      | 8 位      | 8 位      | 8 位         | 8 位         |
| `short`     | 16 位     | 16 位     | 16 位        | 16 位        |
| `int`       | 32 位     | 32 位     | 32 位        | 32 位        |
| `long`      | 32 位     | 64 位     | 32 位        | 32 位        |
| `long long` | 64 位     | 64 位     | 64 位        | 64 位        |

注意 `long` 在 Unix 上随系统架构位数变化，在  Windows 上认为都为 32bit 比较安全，因此跨平台编程时要避免使用 `long`。

## 修饰符确定字面常量

在数字后面追加 `U` 和 `L` 表示不同类型的字面常量，例如 `32L` 等价于 `long`，`32UL` 等价于 `unsigned long`，如果 `int` 装不下会自动选择较大的类型。

## `stdint.h` 标准化

C 语言标准并没有规定 int 就是32位的，只不过主流操作系统一致认为是32位的而已。为了解决不同操作系统上对类型定义混乱的问题，C语言标准引入了 `stdint.h`，包含一系列类型别名(typedef)，这些别名保证不论是什么操作系统什么架构，都是固定的大小，避免跨平台的麻烦

```c
typedef char int8_t;
typedef short int16_t;
typedef int int32_t;
typedef long long int64_t;
```

### `intptr_t` 和 `uintptr_t`：自动随系统位数决定大小

这两个类型是用来指定指针大小的，指针的本质是内存地址，有时候我们需要把指针的地址值存在整型变量里，而 32 位平台上的指针是 32 位，64 位平台上的指针是 64 位，使用自动随系统位数变化的 `intptr_t` 和 `uintptr_t` 即可

### `size_t`：表示大小的整数类型

标准库大量使用的用于表示大小的类型，名字比较直观，主流操作系统上等价于 `uintptr_t` ，同时 `ssize_t` 等价于 `intptr_t` 

## 强制类型转换和扩展

C 语言使用 `(type)x` 把任意类型的 `x` 转换为 `type` 类型，如果原类型比目的类型小，那么根据目的类型是否有符号来自动扩展符号位

### 自动类型提升

- 小类型和大类型做数学运算，返回结果是大类型
- 如果有一个类型是无符号的但是大小不等，则还是选择较大类型
- 如果有一个类型是无符号但大小相等，结果返回无符号的类型

# 浮点数和标准库数学函数

`float` 由 4 个字节组成，最高位是符号位，接着的 8 位是指数位(e)，剩下的 23 位是底数位(m)，值得注意的是指数位(e)是 +127 以后表示的。类似于科学计数法，不过是二进制

## `abs()` 和 `fabs()`：取绝对值

`abs()` 取整数绝对值，返回一个整数，因此不能用来取浮点数绝对值（即使编译通过也不行，因为输入一个浮点类型相当于 `x = (float)abs((int)x)` 被隐式转换成 `int`，调用了 `x = abs(-3)`，并不产生错误），对于浮点数要使用 `fabs()`

## `std::abs()`：自动根据类型重载

C++ 中可以用 `std::abs` 替代 abs，这个在 `std` 命名空间中的版本是带有多种重载的，使用 C++ 建议别用全局的任何函数（C 语言原始的），始终带上 std:: 前缀（C++ 改良后的），这也是为什么不建议用 `using namespace std`

## 指针的概念

本质上只是一个整数，没有类型，有类型只是方便，不知道什么类型用 `void*` 也可以，因为可以捕获任何指针但不能解引用，要转换成具体类型。

指向一个内存中的变量，任何类型 `T` 对应的指针类型是 `T*`，通过 `&` 获取一个变量的地址即指针，通过 `*` 访问指针指向的变量（左值），指针指向了变量，通过指针的 `*` 写入的值，会造成原变量也改变，这正是指针的用法。

```c
int x = 1;
int* p = &x; // x = 1
*p = 2; // x = 2
unsigned long address = (unsigned long)p; // 强制转换地址成整数值
```

在 32bit 系统上指针 `p` 是一个 4byte 整数，是变量 `x` 在内存中位置的第一个字节。

## 二级指针

`T**` 是**指向指针的指针**，用于存储一级指针变量的内存地址

```c
int** q = &p; // 指向 p
```

## 引用

#面试常考

引用和指针是一样的，但是 C++ 的 `int&&` 并不是二级指针，而是右值引用

**引用是指针包装了一层语法糖，区别是：**

- 不需要手动写 `&` 和 `*` 运算符（解引用）
- 无法重新赋值，指向新的变量
- 无法为空指针
- 拷贝是导致他指向的值拷贝，而不是对内存地址号的拷贝

## 指针的应用

**如果要一个函数返回多个变量，可以用指针作为参数**

```c
int func(int* psecond) {
  *psecond = 2;
  return 1;
}
int main() {
  int second;
  int first = func(&second); // *second = 2, *first = 1;
}
```

第一个返回值照常通过 `return` 返回，第二个返回值会写入到调用者提供的指针所指向的变量。指针的作用就在于通过获取调用者变量的地址，让被调用函数也能访问到调用者的变量。

## 空指针和应用

指针可以指向一个变量，也可以什么都不指向即指向 `NULL`，这就是空指针

假如一个参数类 `Params` 非常复杂，然后函数是 `func(Params *pars)`，这样只要给`pars` 传一个空指针，就表示“用户不想指定
这个参数”的意思。C++ 可以用更安全的 `func(std::optional<Params> pars)` 来替代，在下面的例子中用空指针来表示“调用者不需要 `second` 返回”，在 `func` 内还需要通过 `if (psecond)` 来判断是否为空

```c
int func(int* psecond) {
  if (psecond != NULL)
    *psecond = 2;
  return 1;
}
int main() {
  int first = func(NULL);
}
```

### `NULL` 的定义和 `nullptr` (C++11)

#面试常考 

在 C++ 中会直接定义为常数 0，因此可以直接用 `0` 替换 `NULL`，但是可能区分不出 `func(int*)` 和 `func(int)` 的重载，推荐使用 `nullptr`，他的类型是 `std::nullptr_t`，可以隐式转换为任意类型的指针

 C 语言中定义为 `((void*)0)`，因为 C++ 不允许 `void` 指针转换为 `int`，因此 `int* p = (void*)0` 的强制类型转换会编译错误

## 指向数组的指针

数组的元素在内存中连续存储，`&a[0]` 可以简写为 `a`，因为 `T[]` 可以自动转换为 `T*` 指针，指向的是数组中第 0 个元素的地址，**俗称数组的首地址**，因此也可以使用 `*(a + 2)` 访问数组第二个元素

```c
char a[4] = {0, 1, 2, 3};
char* p = &a[0]; // *p = 1 为第0个元素的地址
p = p + 2; // p指向第2个元素的地址
```

### 定长数组可以由首地址指针唯一确定

只要知道了首地址，就可以确定整个数组。这是因为数组的元素都是连续存储的，只要把首地址加上 `n`，就可以得到第 `n` 个元素的地址。从而任意一个元素的地址都可以随意访问。因此对于定长数组，只需要一个指针就可以了。

### 指针加法

给类型为 `T` 的指针加 `n`，实际上是加了 `sizeof(T) * n`，直观理解就是加上 `n` 为访问之后的第 `n` 个元素，但是每个类型占的字节都不一样，C 语言比较智能帮你加上类型大小便于访问了。

如果确实需要让指针的地址加 `1` 而不是指向下一个元素，可以先转换成 `char*`

```c
int* p = &a[0];
p = (int*)((char*)p + 1);
```

### 指针的 `==` 比较

指针的 `==` 比较的是地址是否相等，如果是 `char*` 字符串，这么比较两个字符串是否相等就不行。

```c
char* str1 = "test";
char* str2 = "test";
```

# 动态数组的分配和释放

`char a[4]` 可以在编译期确定一片栈上的连续内存，大小为 4 字节，4 必须是常量。

非编译期常量例如 `char a[n]` 在 gcc 上可以编译通过，因为调用了 gcc 特有的 `alloca(n)` 函数，在当前函数的栈上分配内存，函数退出时也会自动释放。可见栈上动态数组不是标准的 C 语言特性，是无法跨平台使用的，**一般认为栈上的东西都是固定长度的**。

注意动态数组和可变长数组(C99)不是同一个东西，之后补充。

## `malloc()`：在堆上分配内存

#面试常考 

调用 `stdlib.h` 的 `malloc(n)` 就可以从堆上分配一段 `n` 字节的连续内存，这里的 `n` 就可以不是常量了，栈上的内存会在函数退出时自动释放，而堆上的内存不会，需要手动 `free(a)` 释放

`n` 指分配多少字节，而不是分配多少个元素。`malloc` 并不知道你是什么类型的数组，不知道返回以后等待他的是 `(int*)`还是 `(char*)`，他只是分配字节。分配不当导致访问越界现象，轻则程序直接奔溃，重则被利用来入侵系统。因此分配的时候要使用

```c
T *a = (T*)malloc(n * sizeof(T));
```

其中 `sizeof(T)` 来获取任意类型 `T` 的大小，返回的单位是字节。

对于 C 使用 `malloc` 和 `free`，C++ 使用 `new` 和 `delete`，不许混用

## 作为函数参数

动态长度的数组（通过 `malloc` 在堆上分配的）就需要起始地址（指针）和数组长度（`int`）两个，才能确定下来，所以要把动态数组的引用传给函数，需要有两个参数，一个指针加一个长度

```c
void fillarr(char* a, int n) {
  for (int i = 0; i < n; i++) {
    a[i] = 42;
  }
}
int main() {
  int n = rand() % 1024;
  char* a = (char*)malloc(n);
  fillarr(a, n);
  free(a);
}
```

### 函数需要输入和输出数组

仅读取：`void func(const T* data, size_t size);`
仅修改：`void func(T* data, size_t size)`
在函数体内创建数组：`T* func(size_t* psize)`，函数体调用 `malloc()`

### `const` 修饰指针

让调用者清楚哪些函数会修改数组，哪些是只读的。`const char*` 和 `char* const` 第一个修饰的是指针指向的值，第二个修饰的是指针变量本身

## 野指针和空悬指针

#面试常考 #内存管理 

### 野指针(*Wild Pointer*)

野指针是一个未被初始化或已被释放的指针。所以它的值是不确定的，可能指向任意内存地址。访问野指针可能导致未定义行为，如程序崩溃、数据损坏等。

```c
int* makearr() {
	int a[1024];
	for (int i = 0; i < 1024; i++) {
		a[i] = i;
	}
	return a;
}
int main() {
	int* a = makearr();
}
```

`makearr` 中在栈上分配了定长数组 `a[1024]`，在函数 `return` 后就会被释放，但是该函数返回的是 `int*` 指针，即返回的指针必然指向已经释放掉的内存区域，变成**野指针**，外面访问就会出错，如果没有出错还可能会访问到被其他变量覆盖的数据

可以改用 `malloc` 或者 `new` 在堆上分配内存，函数退出时不会释放数组，调用者就可以访问到正确的数据了。当然这样需要调用者在退出时手动调用一下 `free(a)`，因为堆内存不会自动释放

### 空悬指针(*Dangling Pointer*)

空悬指针是指向已经被释放（如删除、回收）的内存的指针。这种指针仍然具有以前分配的内存地址，但是这块内存可能已经被其他对象或数据占用。访问空悬指针同样会导致未定义行为。

```cpp
int *ptr = new int(42);
delete ptr; // 释放内存

// 此时，ptr成为一个空悬指针，因为它指向的内存已经被释放
std::cout << *ptr << std::endl; // 访问空悬指针，可能导致未定义行为
```

为了避免野指针和空悬指针引发的问题，我们应该：

1. 在使用指针前对其进行初始化，如将其初始化为`nullptr`。
2. 在释放指针指向的内存后，将指针设为`nullptr`，避免误访问已释放的内存。
3. 在使用指针前检查其有效性，确保指针指向合法内存。

## C 语言特性：`T[]` 和 `T*`

给函数参数传入一个数组，实际上等同于传入他的首地址指针，本质上属于按引用传递。
`func(int arr[])`, `func(int arr[6])` 等价于 `func(int* arr)`

**为了防止混淆建议全部使用 `func(int* arr)`**。如果真需要传入一个数组而不是数组的首地址指针，用 C++ 的 `std::array<int, 6>`，这是按值传递的，里面的修改不会变化外面的值

### 不要滥用 `sizeof()` 获取数组长度

`sizeof(a)` 只适合获取静态数组，不适合动态数组和函数参数中的数组。因为他们都是个首
地址指针，是指针类型，`sizeof(a)` 获取到的是指针的大小，始终是8 字节

对于动态数组请你手动传入数组长度作为参数，对于 `vector` 请用 `size()` 成员函数

# 字符串

字符串是字符组成的数组，作为参数时会自动转换成 `char*` 即首地址指针

```c
char str = {'a', 'b', 'c' };
printf("%3s\n", str); // 编译期常量，printf 无法知道字符串的长度，所以要在格式里写明
int len = sizeof(str);
printf("%*s\n", len, str); // 额外提供一个运行期变量
```

不指定字符串长度，使用 `%s` 会一直读取字符串直到出现 `\0` 为止，在 Linux 系统还是能正常运行是因为 Linux 默认用 `\0` 填充了栈，windows用 `0xcc` 填充栈，就会出现乱码数字

## `strcpy` 和 `memcpy`

对于一个不定长数组，需要一个首地址指针＋数组长度才能确定下来

`strcpy(dst, src)` 只需要两个指针做参数，而 `memcpy(dst, src, n)` 额外需要一个长度做参数。因为 `strcpy` 针对的是以 `\0` 结尾的字符数组，而 `memcpy` 需要面对的是任意类型的数组


## `strlen` 和 `sizeof`：计算字符串长度

#面试常考 

`strlen` 用于计算 C 风格字符串（以空字符 '\0' 结尾的字符数组）的长度，即不包括结尾的空字符的字符个数

`strlen(s)` 内部实现就是计算第几个字符为 `\0` ，字符串内容加上 `\0` 就是字符串长度了

```c
size_t strlen(const char *str) {
    size_t length = 0;
    while (*str++)
        ++length;
    return length;
}
```

`sizeof` 是一个 C++ 编译期间计算的操作符，用于计算数据类型或对象所占用的字节数，使用 `sizeof` 计算的字符串实际占据的内存比 `strlen` 的长度多 1 字节(要保存 `\0`)

## `scanf` 要指定缓冲区大小

下面的代码期望输入一个长度为 10 的字符串

```c
char str[10];
scanf("%s", str);
```

这个写法非常危险，如果输入了超过长度 10 的字符串，程序会崩溃，最好要写 `scanf("%10s", str)`，然而字符串还含一个 `\0` 所以实际上应该设定长度为 11。在 C++ 中更内存安全的方法是使用 `std::string` 容器和 `cin` 输入流

## 字符串的比较

```c
char* str1 = "test";
char* str2 = "test";
```

由于编译器会把相同的字符串自动放在同样的位置以便节省存储空间，所以两个字符串指针刚好指向了同样的地址。但是不能依赖这个巧合。如果两个字符串是在栈上分配的数组，那么首地址就不
一样，从而始终会返回  `false` 了

```c
char str1[] = "test";
char str2[] = "test";
```

因此请使用 `strcmp()` 比较字符串
